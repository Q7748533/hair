## 6. Site-Wide Standards (post full audit iteration)

### Header (identical on all 13 pages including 404)
```
Home | Straighteners | Hair Dryers | Top Picks | About
```
+ hamburger for mobile, + full mobile-nav overlay with 8 items.

### Footer (identical on all 13 pages)
3 columns: Brand/Tagline + Palm Media | Categories (5 links) | Info (4 links + email)
+ 2 legal lines (Amazon disclaimer + "specs subject to change").

### Cookie banner (Consent Mode v2 compliant)
- Default: `analytics_storage: denied` etc.
- "Accept All" → `gtag('consent','update','granted')` for all 4 scopes
- "Essential Only" → explicit `denied` (actually does nothing tracking-wise) — **not fake anymore**
- Persists via `localStorage.cookie_consent`
- Grants on next visit automatically if previously accepted

### GA4 snippet (identical on all 13 pages)
Inline `<script>` with consent defaults BEFORE the async gtag loader — ensures defaults take effect before any tracking.

### URL structure (cleanUrls: true)
- Canonicals: `https://hairtoolslab.com/foo` (no `.html`)
- Homepage canonical: `https://hairtoolslab.com/` (bare root)
- Internal hrefs: `href="foo"` (relative, no `.html`)
- Homepage href: `href="/"`

### SEO checklist per page
✅ Unique title + description + canonical + robots
✅ og:title + og:description + og:url + og:type + og:site_name + og:image (+width/height/alt)
✅ twitter:card = summary_large_image (never plain "summary"), + twitter:image + twitter:image:alt
✅ Exactly one `<h1>`
✅ Breadcrumb (visible + sometimes in JSON-LD)
✅ JSON-LD schema (see below)

### JSON-LD schema coverage (100%)

8. **cleanUrls + suffix-less canonicals + suffix-less hrefs** — no mixed mode (Google would de-dupe otherwise)
9. **Consent Mode v2 with real "Essential Only"** — fake consent buttons are GDPR risk
