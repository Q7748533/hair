## 6b. Performance Optimizations (applied site-wide, 2026-05-10)

All changes are free wins (zero risk, no UX regression). Verified 13/13 pages.

### Load-order hints (every public page)
```html
<link rel="stylesheet" href="assets/styles.min.css">
<!-- Perf: warm up Amazon CDN connection while HTML parses -->
<link rel="preconnect" href="https://m.media-amazon.com" crossorigin>
<link rel="dns-prefetch" href="https://m.media-amazon.com">
```

### Review pages only: LCP boost for hero image
```html
<!-- Perf: preload LCP hero (parallel with HTML parse) -->
<link rel="preload" as="image" href="https://m.media-amazon.com/images/I/XXX._SL1500_.jpg" fetchpriority="high">
...
<img src="..." ... fetchpriority="high" decoding="async">
```

### GA4 defer strategy (`googletagmanager.com/gtag/js` library, ~70KB)
Consent defaults and `gtag('config')` fire **immediately** inline. The actual library load is wrapped:
```js
(function(){
  var loaded = false;
  function loadGA(){ if(loaded) return; loaded=true;
    var s=document.createElement('script');
    s.async=true;
    s.src='https://www.googletagmanager.com/gtag/js?id=G-HFT4XZ2YB2';
    document.head.appendChild(s);
  }
  if ('requestIdleCallback' in window) requestIdleCallback(loadGA, {timeout: 4000});
  else setTimeout(loadGA, 3000);
  ['scroll','click','keydown','touchstart','mousemove'].forEach(function(ev){
    window.addEventListener(ev, loadGA, {once:true, passive:true});
  });
})();
```
Trade-off: bot visits + sub-3s bounces may not be tracked. Acceptable for this site (real human engagement is what we care about).

### CSS minification
- `styles.css` (source): 20,019 bytes
- `styles.min.css` (prod): 14,062 bytes (-29.8%)
- **Workflow:** edit `styles.css`, then re-minify. The minifier is conservative (strips comments, collapses whitespace, drops last `;` before `}`, preserves strings/url()/content). If adding new rules, just edit the `.css` and re-run a trusted minifier — or use online tool like cssnano.

### What we intentionally did NOT do
- **Vercel Image Optimization** (`/_vercel/image?url=...`) — adds cost, only helps mobile LCP meaningfully. Revisit when GSC shows sustained mobile traffic where Amazon CDN latency matters.
- **AVIF/WebP conversion** — same reason. Amazon images already serve as JPEG with their own CDN caching.
- **Critical CSS inlining** — `styles.min.css` is 14KB; inlining would bloat every HTML and hurt total transfer on multi-page sessions.
- **Service Worker / offline caching** — overkill for a 13-page static site.

## 15. CSS "失效" 假警报 & 缓存教训 (2026-05-10)

Owner 报告 "CSS 失效" 需要修复。完整诊断后确认 **代码本身没有问题**：
- `styles.min.css` 大括号配对完整 (235/235)，无语法错误
- 所有 HTML 使用的 97 个 class 在 CSS 中全部有定义
- `styles.css` 与 `styles.min.css` 的 class 集合一致（仅 `.com` 是 URL 里的假匹配）
- 没有 BOM、没有错误字符、没有未闭合注释

**真正的原因：浏览器缓存**。`vercel.json` 给 `/assets/*` 设置了 `max-age=31536000, immutable`，CSS 更新后老访客会沿用旧版本 1 年。

**Owner 自己 hard-refresh 后确认恢复正常**，未改动任何代码。

### 以后遇到"CSS 失效"类报告的诊断顺序
1. **先问症状**：整页无样式 / 部分错位 / 只在自己浏览器坏 — 三种症状对应三种不同根因
2. **验证代码**：大括号配对、HTML class vs CSS selector 差集、字符编码
3. **验证部署**：`curl -I` 看 CSS 是否返回 200 + 正确 Content-Type
4. **最后才怀疑缓存** — 但 immutable 缓存策略下这是最常见的

### 未来可选增强（owner 当前未采纳，仅记录）
- 给 `<link rel="stylesheet" href="assets/styles.min.css?v=YYYYMMDD">` 加版本查询串，每次改 CSS bump 一次 → 强制所有访客重新下载
- 或者把 `styles.min.css` 改成文件名带 hash（如 `styles.a3f91c.min.css`），永久解决缓存不一致问题
- **取舍**：加版本号会让每次 CSS 更新之后老访客触发一次额外下载，但能彻底杜绝"owner 改了 CSS 却看不到变化"这类问题。当前 1 年 immutable 缓存对首次加载性能最优，但开发-部署摩擦较大

### 教训
- 报告"CSS 失效"时 **不要立即改代码** — 先排查是否只是缓存或部署延迟
- `immutable` 缓存 + 固定文件名 是典型陷阱；大型项目通常配合 build hash 解决
- DevTools "Disable cache" + Cmd/Ctrl+Shift+R 是开发时的必备习惯
