## 2. Tech Stack & Deployment

- **Static HTML + CSS** (no JS framework)
- **Shared stylesheet:** `assets/styles.css` (source, 20KB) + `assets/styles.min.css` (prod, 14KB, -30%)
  - All HTML references `styles.min.css`. Never edit `.min.css` directly вҖ” edit `.css` and re-minify.
- **Shared JS:** `assets/site.min.js` (GA4 + Consent Mode + Cookie Banner + Scroll UI + Mobile Nav)
  - All HTML references this via `<script src="assets/site.min.js" defer></script>` in `<head>`
  - Inline JS has been fully eliminated from all pages
- **CSS build pipeline:** `.github/workflows/build-css.yml` вҖ” auto-minifies `styles.css` вҶ’ `styles.min.css` on push
- **Deploy:** Vercel, auto-deploy from `main` branch
- **Config:** `vercel.json` вҶ’ `cleanUrls: true`, `trailingSlash: false`, cache headers, /landing redirects
- **Analytics:** GA4, ID `G-HFT4XZ2YB2`, Consent Mode v2 (default deny until user opts in)
  - gtag.js library is **deferred**: consent defaults fire immediately, but the 70KB library only loads on `requestIdleCallback` (timeout 4s) or first user interaction (scroll/click/keydown/touchstart/mousemove).
- **Images:** Direct Amazon CDN URLs (no local hosting), hotlinked for og:image too
- **Perf hints:** preconnect + dns-prefetch to `m.media-amazon.com` on every page. Review pages additionally preload the hero `<img>` with `fetchpriority="high"` and `decoding="async"`.
- **Internal docs NOT shipped to prod:** `.vercelignore` excludes `.kiro-memory.md`, `soul.ilang`, `project-rules.md`, `ilang-example.md`, `data/`

## 6b. Performance Optimizations (applied site-wide, 2026-05-10)

All changes are free wins (zero risk, no UX regression). Verified 13/13 pages.

### Load-order hints (every public page)
```html
<link rel="stylesheet" href="assets/styles.min.css">
<!-- Perf: warm up Amazon CDN connection while HTML parses -->
<link rel="preconnect" href="https://m.media-amazon.com" crossorigin>
<link rel="dns-prefetch" href="https://m.media-amazon.com">
```

### Review pages only: LCP boost for hero image
```html
<!-- Perf: preload LCP hero (parallel with HTML parse) -->
<link rel="preload" as="image" href="https://m.media-amazon.com/images/I/XXX._SL1500_.jpg" fetchpriority="high">
...
<img src="..." ... fetchpriority="high" decoding="async">
```

### GA4 defer strategy (`googletagmanager.com/gtag/js` library, ~70KB)
Consent defaults and `gtag('config')` fire **immediately** inline. The actual library load is wrapped:
```js
(function(){
  var loaded = false;
  function loadGA(){ if(loaded) return; loaded=true;
    var s=document.createElement('script');
    s.async=true;
    s.src='https://www.googletagmanager.com/gtag/js?id=G-HFT4XZ2YB2';
    document.head.appendChild(s);
  }
  if ('requestIdleCallback' in window) requestIdleCallback(loadGA, {timeout: 4000});
  else setTimeout(loadGA, 3000);
  ['scroll','click','keydown','touchstart','mousemove'].forEach(function(ev){
    window.addEventListener(ev, loadGA, {once:true, passive:true});
  });
})();
```
Trade-off: bot visits + sub-3s bounces may not be tracked. Acceptable for this site (real human engagement is what we care about).

### CSS minification
- `styles.css` (source): 20,019 bytes
- `styles.min.css` (prod): 14,062 bytes (-29.8%)
- **Workflow:** edit `styles.css`, then re-minify. The minifier is conservative (strips comments, collapses whitespace, drops last `;` before `}`, preserves strings/url()/content). If adding new rules, just edit the `.css` and re-run a trusted minifier вҖ” or use online tool like cssnano.

### What we intentionally did NOT do
- **Vercel Image Optimization** (`/_vercel/image?url=...`) вҖ” adds cost, only helps mobile LCP meaningfully. Revisit when GSC shows sustained mobile traffic where Amazon CDN latency matters.
- **AVIF/WebP conversion** вҖ” same reason. Amazon images already serve as JPEG with their own CDN caching.
- **Critical CSS inlining** вҖ” `styles.min.css` is 14KB; inlining would bloat every HTML and hurt total transfer on multi-page sessions.
- **Service Worker / offline caching** вҖ” overkill for a 13-page static site.
